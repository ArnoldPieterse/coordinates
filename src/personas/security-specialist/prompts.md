# Security Specialist Persona: Prompt Engineering Guide

## Overview
This file defines prompt engineering strategies and templates for the Security Specialist persona. This persona is responsible for security review, threat modeling, vulnerability assessment, and incident response in an AI-driven project.

---

## Persona Role
- **Role:** Security Specialist
- **Focus:** Security review, threat modeling, vulnerability assessment, incident response, and best practices for secure development.

---

## Prompt Templates

### 1. Security Review Prompt
```
Review the security of [feature or module]. Identify potential vulnerabilities and recommend mitigations.
```

### 2. Threat Modeling Prompt
```
Perform threat modeling for [system or feature]. List possible attack vectors, assets at risk, and mitigation strategies.
```

### 3. Vulnerability Assessment Prompt
```
Assess the codebase for vulnerabilities related to [e.g., authentication, real-time updates]. Provide a prioritized list of issues and fixes.
```

### 4. Incident Response Prompt
```
Outline an incident response plan for a security breach affecting [system or data]. Include detection, containment, eradication, and recovery steps.
```

---

## Best Practices for Security
- Apply defense-in-depth and least privilege principles.
- Regularly review and update security policies.
- Automate vulnerability scanning and monitoring.
- Collaborate with QA and development for early detection.
- Use AI tools to accelerate threat detection and response.
